# 用户提问示例与推荐响应方式

## 示例 1

用户提问：
- 帮我快速总结这个项目，告诉我用了什么技术栈和目录怎么组织。

Skill 响应方式：
- 使用“快速项目摘要”范式。
- 输出：技术栈、工程化、目录职责、入口路由、状态/API、风险与待确认项。

## 示例 2

用户提问：
- 这个项目 API 是怎么封装的？如果我要新增一个接口，应该放在哪？

Skill 响应方式：
- 使用“API 专题分析”范式。
- 输出当前封装、关键文件、推测设计意图、新增接口标准步骤、风险建议。

## 示例 3

用户提问：
- 全局状态现在是怎么管理的？我后续要加一个用户偏好状态。

Skill 响应方式：
- 使用“状态管理专题分析”范式。
- 输出状态分层、持久化策略、新增状态流程、潜在风险。

## 示例 4

用户提问：
- 请输出完整的 PROJECT_ARCHITECTURE.md，后面我会交给其他 AI 使用。

Skill 响应方式：
- 直接套用 `project-architecture-template.md` 完整章节。
- 所有结论均附证据路径；无法确认项列入“待确认”。

## 示例 5

用户提问：
- 先不要改代码，只帮我识别是否有技术债。

Skill 响应方式：
- 显式声明“只读模式已启用”。
- 基于证据输出技术债观察，使用谨慎措辞：
  - “从当前代码推测”
  - “待确认”
