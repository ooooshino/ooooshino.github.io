## 1. 项目概述

### 1.1 项目定位

- 业务目标：以 Web 单页形式展示个人简历信息（个人信息、技能、工作经历、项目经历、优势总结）。
- 主要用户：招聘方、面试官、合作方、个人主页访客。
- 核心功能域：内容展示与排版，不涉及复杂交互业务流。

## 18. 已知风险与待确认项

### 18.1 已知风险（基于代码证据）
- 风险 1：`CompanyExperience.vue` 存在大段相似内容重复，后续维护易出现多处同步不一致。
- 风险 2：`style.postcss` 使用大量全局标签规则，项目规模扩大后可能出现样式耦合与覆盖冲突。

### 18.2 待确认项
- 待确认 1：是否计划从“单页简历”演进为“多页面个人站点”。
- 待确认 2：是否会引入动态数据（API）和后台管理入口。

### 18.3 后续建议
- 建议 1：若持续扩展内容，优先做“数据结构化 + 模板化渲染”，降低重复编辑成本。
- 建议 2：若引入业务功能，先补齐 router/services/store 的基础工程骨架，再迭代功能。
